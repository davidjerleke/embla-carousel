# Plugins

You can **extend Embla Carousel** with additional features by using **plugins**. A complete list of official plugins is available [here](/plugins/).

---

## Installation

All **official plugins** are published as separate **NPM packages**. Each package name starts with the `embla-carousel` prefix, followed by the plugin's unique name.
For example, to install the `Autoplay` plugin:

<Tabs groupId={TABS_PACKAGE_MANAGER.GROUP_ID}>
  <TabsItem tab={TABS_PACKAGE_MANAGER.TABS.CDN}>
    <PrismHighlight language="html" code={pluginsInstallation_cdn_01} />
  </TabsItem>
  <TabsItem tab={TABS_PACKAGE_MANAGER.TABS.NPM}>
    <PrismHighlight language="shell" code={pluginsInstallation_npm_01} />
  </TabsItem>
  <TabsItem tab={TABS_PACKAGE_MANAGER.TABS.YARN}>
    <PrismHighlight language="shell" code={pluginsInstallation_yarn_01} />
  </TabsItem>
</Tabs>

## Usage

The Embla Carousel **constructor** accepts an **array of plugins**. Each plugin can provide its own [options](/api/plugins/#constructor-options), [methods](/api/plugins/#calling-methods), and [events](/api/plugins/#adding-event-listeners).

### Adding a plugin

The constructor's plugin array is the standard way to add plugins to Embla Carousel. In the example below, the [`Autoplay`](/plugins/autoplay/) plugin is added to the carousel:

<Tabs groupId={TABS_LIBRARY.GROUP_ID}>
  <TabsItem tab={TABS_LIBRARY.TABS.VANILLA}>
    <PrismHighlight language="js" code={addingPlugin_module_01} highlight="7" />
    <PrismHighlight language="html" code={addingPlugin_html_01} />
  </TabsItem>
  <TabsItem tab={TABS_LIBRARY.TABS.REACT}>
    <PrismHighlight language="jsx" code={addingPlugin_react_01} highlight="6" />
  </TabsItem>
  <TabsItem tab={TABS_LIBRARY.TABS.VUE}>
    <PrismHighlight language="html" code={addingPlugin_vue_01} highlight="6" />
  </TabsItem>
  <TabsItem tab={TABS_LIBRARY.TABS.SOLID}>
    <PrismHighlight language="jsx" code={addingPlugin_solid_01} highlight="8" />
  </TabsItem>
  <TabsItem tab={TABS_LIBRARY.TABS.SVELTE}>
    <PrismHighlight
      language="svelte"
      code={addingPlugin_svelte_01}
      highlight="6,18"
    />
  </TabsItem>
</Tabs>

<Admonition>
  Note that you can **update the plugins** passed to the Embla Carousel
  constructor **after initialization** using the
  [`reInit`](/api/methods/#reinit) method.
</Admonition>

### Global options

All [official plugins](/plugins/) allow you to set **global options** that apply to every carousel instance. This lets you override the default plugin settings with your own preferences:

<Tabs groupId={TABS_LIBRARY.GROUP_ID}>
  <TabsItem tab={TABS_LIBRARY.TABS.VANILLA}>
    <PrismHighlight
      language="js"
      code={globalOptions_module_01}
      highlight="4"
    />
    <PrismHighlight language="html" code={addingPlugin_html_01} />
  </TabsItem>
  <TabsItem tab={TABS_LIBRARY.TABS.REACT}>
    <PrismHighlight
      language="jsx"
      code={globalOptions_react_01}
      highlight="5"
    />
  </TabsItem>
  <TabsItem tab={TABS_LIBRARY.TABS.VUE}>
    <PrismHighlight language="html" code={globalOptions_vue_01} highlight="6" />
  </TabsItem>
  <TabsItem tab={TABS_LIBRARY.TABS.SOLID}>
    <PrismHighlight
      language="jsx"
      code={globalOptions_solid_01}
      highlight="5"
    />
  </TabsItem>
  <TabsItem tab={TABS_LIBRARY.TABS.SVELTE}>
    <PrismHighlight
      language="svelte"
      code={globalOptions_svelte_01}
      highlight="5"
    />
  </TabsItem>
</Tabs>

<Admonition type="warning">
  Make sure to assign global options **before** initializing any carousel and
  **only once**. Reassigning global options may cause unexpected behavior and
  lead to confusing code.
</Admonition>
