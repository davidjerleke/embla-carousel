# Events

Embla Carousel provides a set of **events** you can listen to for reacting to **carousel updates and user interactions**.

---

## Usage

Events are available only on an **initialized carousel instance**. They fire throughout the carousel's lifecycle, and any added listeners remain active even after a hard reset with the [`reInit`](/api/methods/#reinit) method.

### Cancelling events

Some events are **cancellable**, allowing you to prevent Embla Carousel's internal logic from running.
To cancel an event, return `false` from the event handler. Returning any other value—such as `true` or `undefined`—will allow the event to proceed normally.

See the [event reference](#reference) to check which events are cancellable.

This code prevents pointer events (dragging) on the carousel whenever the `allowPointerDownEvent` variable is `false`.

### Event data

The event handler receives the event data as the **second argument**, providing useful information about the event. It includes:

- **`type`** — the name of the event that was triggered.
- **`detail`** — event-specific data. See the [event reference](#reference) for details on the data available for each event.

## Reference

Below follows an exhaustive **list of all** Embla Carousel **events** together with information about how they work.

---

### reinit

<ApiMetaData firstRow={['Detail', 'null']} secondRow={['Cancellable', 'no']} />
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('reinit', (emblaApi) => {})`}
/>

Runs when the [`reInit`](/api/methods/#reinit) method is called. When the container or any slide sizes change, or when slides are added/removed, Embla Carousel automatically calls the [`reInit`](/api/methods/#reinit) method which will also fire this event.

---

### destroy

<ApiMetaData firstRow={['Detail', 'null']} secondRow={['Cancellable', 'no']} />
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('destroy', (emblaApi) => {})`}
/>

Runs when the carousel has been destroyed using the [`destroy`](/api/methods/#destroy) method. This only fires once and will be the last event the carousel fires.

---

### select

<ApiMetaData
  firstRow={['Detail', 'targetSnap: number, sourceSnap: number']}
  secondRow={['Cancellable', 'no']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('select', (emblaApi, event) => {
  const { targetSnap, sourceSnap } = event.detail
})`}
/>

Runs when the selected scroll snap changes. The select event is triggered by drag interactions or the [`scrollToNext`](/api/methods/#scrolltonext), [`scrollToPrev`](/api/methods/#scrolltoprev) or[`scrollToSnap`](/api/methods/#scrolltosnap) methods.

---

### scroll

<ApiMetaData
  firstRow={['Detail', 'isDragging: boolean']}
  secondRow={['Cancellable', 'no']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('scroll', (emblaApi, event) => {
  const { isDragging } = event.detail
})`}
/>

Runs when the carousel is scrolling. It might be a good idea to throttle this if you're doing expensive stuff in your callback function.

---

### settle

<ApiMetaData firstRow={['Detail', 'null']} secondRow={['Cancellable', 'no']} />
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('settle', (emblaApi) => {})`}
/>

Runs when the carousel has settled after scroll has been triggered. Please note that this can take longer than you think when [`dragFree`](/api/options/#dragfree) is enabled or when using slow [`durations`](/api/options/#duration).

---

### resize

<ApiMetaData
  firstRow={['Detail', 'ResizeObserverEntry[]']}
  secondRow={['Cancellable', 'yes']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('resize', (emblaApi, event) => {
  const entries = event.detail
})`}
/>

Runs when the carousel container or the slide sizes change. It's using [ResizeObserver](https://developer.mozilla.org/en-US/docs/Web/API/ResizeObserver) under the hood.

---

### slidesinview

<ApiMetaData
  firstRow={['Detail', 'IntersectionObserverEntry[]']}
  secondRow={['Cancellable', 'no']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('slidesinview', (emblaApi, event) => {
  const entries = event.detail
})`}
/>

Runs when any slide has **entered** or **exited** the viewport. This event is intended to be used together with the [`slidesInView`](/api/methods/#slidesinview) and/or [`slidesNotInView`](/api/methods/#slidesnotinview) methods.

---

### slideschanged

<ApiMetaData
  firstRow={['Detail', 'MutationRecord[]']}
  secondRow={['Cancellable', 'yes']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('slideschanged', (emblaApi, event) => {
  const mutationRecords = event.detail
})`}
/>

Runs when slides are added to, or removed from the carousel [container](/api/options/#container). It's using [MutationObserver](https://developer.mozilla.org/en-US/docs/Web/API/MutationObserver) under the hood.

---

### slidefocus

<ApiMetaData
  firstRow={['Detail', 'FocusEvent']}
  secondRow={['Cancellable', 'yes']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('slidefocus', (emblaApi, event) => {
  const focusEvent = event.detail
})`}
/>

Runs when a slide receives focus. For example, when a focusable element like a button, link or input receives focus inside a slide.

---

### pointerdown

<ApiMetaData
  firstRow={['Detail', 'TouchEvent | MouseEvent']}
  secondRow={['Cancellable', 'yes']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('pointerdown', (emblaApi, event) => {
  const pointerEvent = event.detail
})`}
/>

Runs when the user has a pointer down on the carousel. It's triggered by a `touchstart` or a `mousedown` event.

---

### pointermove

<ApiMetaData
  firstRow={['Detail', 'TouchEvent | MouseEvent']}
  secondRow={['Cancellable', 'yes']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('pointermove', (emblaApi, event) => {
  const pointerEvent = event.detail
})`}
/>

Runs when the user is moving the pointer on the carousel. It's triggered by a `touchmove` or a `mousemove` event.

---

### pointerup

<ApiMetaData
  firstRow={['Detail', 'TouchEvent | MouseEvent']}
  secondRow={['Cancellable', 'no']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('pointerup', (emblaApi, event) => {
  const pointerEvent = event.detail
})`}
/>

Runs when the user has released the pointer from the carousel. It's triggered by a `touchend` or a `mouseup` event.

---

### scrolloptimize

<ApiMetaData
  firstRow={['Detail', 'inViewSlides: number[], leftViewSlides: number[]']}
  secondRow={['Cancellable', 'yes']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('scrolloptimize', (emblaApi, event) => {
  const { inViewSlides, leftViewSlides } = event.detail
})`}
/>

Runs when the carousel shifts GPU acceleration to optimize scrolling performance. This event is triggered with an in view offset to ensure smooth scrolling performance when slides are entering or leaving the viewport.
