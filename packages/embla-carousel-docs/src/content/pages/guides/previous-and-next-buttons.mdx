# Prev & Next Buttons

This guide shows you how to **add previous and next buttons** to control carousel navigation in Embla Carousel.

---

## Prerequisites

- [How Embla Carousel Works](/guides/how-embla-carousel-works)
- [Required Setup](/guides/required-setup)

## Button placement

If your carousel is [`draggable`](/api/options/#draggable), the **root node** — the element passed to the `EmblaCarousel` initializer (e.g., `.embla__viewport`) — responds to pointer events. To avoid **unintended drag interactions** when users click them, **place your navigation buttons outside the root element**, like this:

## Adding buttons

Adding navigation buttons is simple. Create your button elements and connect them to Embla's navigation methods — [`goToPrev`](/api/methods/#goToPrev) and [`goToNext`](/api/methods/#goToNext).

These methods let you scroll the carousel programmatically when users click the buttons, giving them an intuitive way to move between slides.

## Buttons state

We'll enhance the user experience by **disabling the navigation buttons** when the carousel can't scroll any further.

For non-looping carousels, it's good UX to indicate when the navigation buttons can't be used — for example, when you're already at the first or last slide.

We'll handle this by toggling each button's state between **enabled** and **disabled**, updating them when:

- The [`select`](/api/events/#select) event fires — when the selected scroll snap changes.
- The [`reinit`](/api/events/#reinit) event fires — when the carousel resets.

## Notes

- **Button placement:** Place the prev/next buttons **outside** the Embla root element (the one passed to the initializer, e.g. `.embla__viewport`) to prevent accidental drag interactions.
- **Looping carousels:** In **looping** carousels with enough slides to wrap around, the prev/next methods always trigger scrolling — buttons are never disabled.
- **No-op behavior:** On **non-looping** carousels, calling [`goToPrev`](/api/methods/#goToPrev) or [`goToNext`](/api/methods/#goToNext) when already at the first or last slide has no effect. This is normal and ensures consistent, predictable behavior.
