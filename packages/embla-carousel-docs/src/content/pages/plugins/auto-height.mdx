## Installation

First you need to install the **npm package** and save it to your dependencies:

<Tabs groupId={TABS_PACKAGE_MANAGER.GROUP_ID}>
  <TabsItem tab={TABS_PACKAGE_MANAGER.TABS.CDN}>

    ```html
    <script src="https://unpkg.com/embla-carousel-auto-height/embla-carousel-auto-height.umd.js"></script>
    ```

  </TabsItem>
  <TabsItem tab={TABS_PACKAGE_MANAGER.TABS.NPM}>

    ```shell
    npm install embla-carousel-auto-height --save
    ```

  </TabsItem>
  <TabsItem tab={TABS_PACKAGE_MANAGER.TABS.YARN}>

    ```shell
    yarn add embla-carousel-auto-height
    ```

  </TabsItem>
</Tabs>

<Admonition type="note">

You can make use of CSS transitions to **transition height** changes. But beware: Transitioning height triggers reflow and may cause a performance hit.

<br />

<PrismHighlight
  language="css"
  hideLabel
  code={`.embla__container {
  transition: height 0.2s;
}`}
/>

</Admonition>

<Admonition type="warning">

You want to make sure that each **slide height** is **determined** by the **content** it holds. Add the following to your CSS to achieve this:

<br />

<PrismHighlight
  language="css"
  hideLabel
  highlight="3"
  code={`.embla__container {
  display: flex;
  align-items: flex-start; /* Add this */
}`}
/>

</Admonition>

## Options

Below follows an exhaustive **list of all** `Autoplay` **options** and their default values.

---

### active

<ApiMetaData firstRow={['Type', 'boolean']} secondRow={['Default', 'true']} />
<PrismHighlight language="ts" hideLabel code={`{ active: false | true }`} />

Setting this to `false` will not activate or deactivate the plugin. Useful when used together with the [`breakpoints`](/plugins/auto-height/#breakpoints) option to toggle the plugin active/inactive depending on media queries.

---

### breakpoints

<ApiMetaData
  firstRow={['Type', 'AutoHeightOptionsType']}
  secondRow={['Default', '{}']}
/>
<PrismHighlight
  language="ts"
  hideLabel
  code={`{ 
  active: false,
  '(min-width: 768px)': { active: true }
}`}
/>

An object with options that will be applied for a given breakpoint by overriding the options at the root level.

<Admonition type="note">
  **Note:** If multiple queries match, they will be merged. And when breakpoint
  options clash, the last one in the list has precedence.
</Admonition>

---

### heightEvent

<ApiMetaData firstRow={['Type', 'string']} secondRow={['Default', 'select']} />
<PrismHighlight
  language="ts"
  hideLabel
  code={`{ heightEvent: 'select' | 'slidesinview' }`}
/>

By default, the carousel container's height is adjusted based on the slides in the **selected scroll snap**. Set this option to `'slidesinview'` to adjust the height dynamically according to **all slides currently in view**.
