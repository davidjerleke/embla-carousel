# Dot Buttons

This guide shows you how to add **dot buttons** to control carousel navigation in Embla Carousel.

---

## Prerequisites

- [How Embla Carousel Works](/docs/v8/guides/how-embla-carousel-works)
- [Required Setup](/docs/v8/guides/required-setup)
- [Grouping Slides](/docs/v8/guides/grouping-slides)

## Button placement

If your carousel is draggable, meaning [`watchDrag`](/docs/v8/api/options/#watchdrag) is set to `true`, the **root node** — the element passed to the `EmblaCarousel` initializer (e.g., `.embla__viewport`) — responds to pointer events. To avoid **unintended drag interactions** when users click them, **place your navigation buttons outside the root element**, like this:

## Adding buttons

Embla Carousel exposes the total number of **snap points** — one for each group of slides — through the [`scrollSnapList`](/docs/v8/api/methods/#scrollsnaplist) method. You can use this to **generate a dot button for each snap** dynamically.

Each button should be connected to Embla's [`scrollTo`](/docs/v8/api/methods/#scrollto) method, which scrolls the carousel to a specific snap index. When users click a dot, the carousel will smoothly move to the corresponding slide group, providing an intuitive navigation experience.

## Buttons state

To improve the user experience, let's **add visual feedback** to show which dot button corresponds to the currently selected snap.

You can achieve this by toggling each button's state between **selected** and **not selected**, and keeping them in sync whenever the carousel updates. Update the buttons whenever:

- The [`select`](/docs/v8/api/events/#select) event fires — when the active scroll snap changes.
- The [`reInit`](/docs/v8/api/events/#reinit) event fires — when the carousel reinitializes.

## Notes

- **Button placement:** Place the previous/next buttons **outside** the Embla root element (the one passed to the initializer, e.g., `.embla__viewport`) to avoid accidental drag interactions when users click them.
- **Loop behavior:** When the carousel has `loop` enabled, [`scrollTo`](/docs/v8/api/methods/#scrollto) will automatically choose the **shortest path** to the target snap, scrolling forward or backward as needed.
- **No-op behavior:** Calling [`scrollTo`](/docs/v8/api/methods/#scrollto) when the carousel is already at the given snap has **no effect**. This is expected and ensures navigation remains predictable and consistent.
