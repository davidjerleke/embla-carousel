# Events

Embla Carousel provides a set of **events** you can listen to for reacting to **carousel updates and user interactions**.

---

## Usage

Events are available only on an **initialized carousel instance**. They fire throughout the carousel's lifecycle, and any added listeners remain active even after a hard reset with the [`reInit`](/docs/api/methods/#reinit) method.

After initializing the carousel, you can **subscribe** to events. The following example shows how to listen for the [`slidesInView`](#slidesinview) event:

## Reference

Below follows an exhaustive **list of all** Embla Carousel **events** together with information about how they work.

---

### init

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('init', (emblaApi) => {})`}
/>

Runs when the carousel mounts for the first time. This only fires once which means that it won't fire when the carousel is re-initialized using the [`reInit`](/docs/v8/api/methods/#reinit) method.

---

### reInit

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('reInit', (emblaApi) => {})`}
/>

Runs when the [`reInit`](/docs/v8/api/methods/#reinit) method is called. When the window is resized, Embla Carousel automatically calls the [`reInit`](/docs/v8/api/methods/#reinit) method which will also fire this event.

---

### destroy

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('destroy', (emblaApi) => {})`}
/>

Runs when the carousel has been destroyed using the [`destroy`](/docs/v8/api/methods/#destroy) method. This only fires once and will be the last event the carousel fires.

---

### select

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('select', (emblaApi) => {})`}
/>

/>

Runs when the selected scroll snap changes. The select event is triggered by drag interactions or the [`scrollNext`](/docs/v8/api/methods/#scrollnext), [`scrollPrev`](/docs/v8/api/methods/#scrollprev) or [`scrollTo`](/docs/v8/api/methods/#scrollto) methods.

---

### scroll

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('scroll', (emblaApi) => {})`}
/>

Runs when the carousel is scrolling. It might be a good idea to throttle this if you're doing expensive stuff in your callback function.

---

### settle

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('settle', (emblaApi) => {})`}
/>

Runs when the carousel has settled after scroll has been triggered. Please note that this can take longer than you think when [`dragFree`](/docs/v8/api/options/#dragfree) is enabled or when using slow [`transitions`](/docs/v8/api/options/#duration).

---

### resize

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('resize', (emblaApi) => {})`}
/>

Runs when the carousel container or the slide sizes change. It's using [ResizeObserver](https://developer.mozilla.org/en-US/docs/Web/API/ResizeObserver) under the hood.

---

### slidesInView

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('slidesInView', (emblaApi) => {})`}
/>

/>

Runs when any slide has **entered** or **exited** the viewport. This event is intended to be used together with the [`slidesInView`](/docs/v8/api/methods/#slidesinview) and/or [`slidesNotInView`](/docs/v8/api/methods/#slidesnotinview) methods.

---

### slidesChanged

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('slidesChanged', (emblaApi) => {})`}
/>

Runs when slides are added to, or removed from the carousel [`container`](/docs/v8/api/options/#container). It's using [MutationObserver](https://developer.mozilla.org/en-US/docs/Web/API/MutationObserver) under the hood.

---

### slideFocus

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('slideFocus', (emblaApi) => {})`}
/>

Runs when a slide receives focus. For example, when a focusable element like a button, link or input receives focus inside a slide.

---

### pointerDown

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('pointerDown', (emblaApi) => {})`}
/>

Runs when the user has a pointer down on the carousel. It's triggered by a `touchstart` or a `mousedown` event.

---

### pointerUp

<PrismHighlight
  language="ts"
  hideLabel
  code={`emblaApi.on('pointerUp', (emblaApi) => {})`}
/>

Runs when the user has released the pointer from the carousel. It's triggered by a `touchend` or a `mouseup` event.

---
